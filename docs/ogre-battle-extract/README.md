# Ogre Battle: March of the Black Queen — gameplay extract

**Source:** `tmp-ob-rules.txt` (Sulpher v9.8 FAQ, 2003/2020). The full
file is ~18k lines: the front matter, version notes, the entire
Castle-of-Warren-through-endgame walkthrough, and a checklist
section. None of that is in scope here.

**In scope:** lines 173–3577 — the rules-and-systems sections of the
FAQ:

- Character Stats (s0002)
- Classes (s0003) — the bulk, ~2,400 lines of class entries
- The Opinion Leader (s0004)
- Battle Tactics (s0005)
- Tarot Cards (s0006)
- Things To Understand (s0007)

**Purpose:** extract the gameplay/character mechanics in a form that
can feed future design memos (e.g. follow-ons to
`design-memo-mechanics-research.md`, which already cites OB).
Editorial — not a faithful transcription. Where a mechanic is repeated
across 30+ class entries (e.g. "gain 5–7 HP per level"), it's compressed
into a pattern.

## Notes

- [01 — Stats and deeds](01-stats-and-deeds.md) — the six stats, and
  the two (CHA, ALI) that mutate through play.
- [02 — Class progression](02-class-progression.md) — family trees,
  the gating recipe (Level + CHA + ALI + sometimes Item), item-triggered
  evolutions, alignment-band branches.
- [03 — Combat loop](03-combat-loop.md) — front/back row, attack
  count as the real lever, six damage elements, terrain, day/night.
- [04 — Opinion Leader and Tarot](04-opinion-leader-and-tarot.md) —
  the Lord (a special unit), the tarot questionnaire that picks his
  attack set, and the tarot deck (dual liberation/battle purpose).
- [05 — Design hooks](05-design-hooks.md) — the editorial pass: what
  in here is worth lifting into the current game for complexity,
  progression, and forward motion.

## What this isn't

- Not a walkthrough digest. Maps, scenarios, treasure locations,
  shop listings, special character recruitment quests — all skipped.
- Not a full class catalog. 60+ classes; representative examples
  per family tree, not every stat block. Read the source if you need
  the exact numbers for any one class.
- Not balance numbers as-is. OB's HP/STR scales and SNES-era
  AGI/INT ranges are not directly portable — these notes lift
  _structure_, not values.
